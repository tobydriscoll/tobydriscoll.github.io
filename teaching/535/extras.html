Writing up mathematics on the computer

If you continue in a technical career, chances are good that you will need to typeset (i.e., word-process) a document with mathematics in it. There are a number of tools around for this. In decreasing order of laziness (and generally increasing quality), they are
  • Microsoft Word: Gets something mostly legible on the page with minimum fuss.
  • Openoffice: Free Office replacment for many platforms. Decent math editor that lets you type formulas or click buttons and menus for them.
  • TeXmacs: Free, quirky WYSIWYG editor that produces much nicer looking stuff in a fairly good GUI environment. Takes some time to get used to, though.
  • LaTeX: What the pros use. Produces the best-looking output by far and can handle very complex math. However, it is more of a programming language than a traditional word processor. There are some integrated editors out there to make it a lot easier: WinEdt (Windows, shareware) and Kile (Linux, free) are two of the best.

MATLAB demos for the course

To run a demo:

  1. Download the file(s) to your hard drive (you may want to create a directory/folder).
  2. Start MATLAB.
  3. Use the "Change Directory" field near the top of the MATLAB window to type in the directory from step 1 (or use the adjacent button to surf there).
  4. Type the name of the .m file (without extension) at the ">>" prompt.

  • Convergence of Fourier partial sums: Fouriersummation.m and Fouriersummation.fig (both files needed to run)
  • Time evolution in the heat equation: get heatdemo.m and heatdemo.fig.
  • Eigenfunctions on the unit disk: Get and run besseldemo.m
